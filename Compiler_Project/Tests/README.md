# اختبارات مترجم اللغة البرمجة العربية

## نظرة عامة

يحتوي هذا المجلد على جميع الاختبارات لمشروع مترجم اللغة البرمجة العربية. تم تنظيم الاختبارات في فئات مختلفة لضمان جودة عالية وتغطية شاملة لجميع مكونات المشروع.

## هيكل الاختبارات

### 1. اختبارات الوحدة (UnitTests)
- **الهدف**: اختبار كل مكون من مكونات المترجم بشكل منفصل
- **المكونات المختبرة**:
  - المحلل المعجمي (Lexer)
  - المحلل النحوي (Parser)
  - المحلل الدلالي (Semantic Analyzer)
  - مولد الكود (Code Generator)
  - شجرة البنية المجردة (AST)
  - شجرة الإعراب (Parse Tree)
  - جدول الرموز (Symbol Table)

### 2. اختبارات التكامل (IntegrationTests)
- **الهدف**: اختبار التفاعل بين مكونات المترجم المختلفة
- **الاختبارات**:
  - التكامل بين المحلل المعجمي والنحوي
  - التكامل بين المحلل النحوي والدلالي
  - التكامل بين المحلل الدلالي ومولد الكود
  - التكامل بين جميع المكونات

### 3. اختبارات النظام (SystemTests)
- **الهدف**: اختبار المترجم ككل مع واجهة المستخدم
- **الاختبارات**:
  - اختبار الترجمة الكاملة للبرامج
  - اختبار واجهة المستخدم
  - اختبار حفظ وتحميل الملفات
  - اختبار عرض النتائج

### 4. اختبارات الأداء (PerformanceTests)
- **الهدف**: قياس وتحسين أداء المترجم
- **الاختبارات**:
  - قياس سرعة الترجمة
  - قياس استخدام الذاكرة
  - اختبار البرامج الكبيرة
  - تحليل الاختناقات

### 5. حالات الاختبار (TestCases)
- **الهدف**: مجموعة من البرامج العربية للاختبار
- **الأنواع**:
  - برامج صحيحة نحوياً ودلالياً
  - برامج تحتوي على أخطاء معجمية
  - برامج تحتوي على أخطاء نحوية
  - برامج تحتوي على أخطاء دلالية
  - برامج معقدة تستخدم جميع الميزات

## كيفية تشغيل الاختبارات

### 1. متطلبات التشغيل
- Visual Studio 2015
- .NET Framework 4.5.2
- C++ Compiler (C++11)
- Unity Test Framework (لاختبارات C#)

### 2. تشغيل اختبارات C
```bash

make test

# أو تشغيل الاختبارات الفردية
./test_lexer
./test_parser
./test_semantic
./test_codegen
```

```bash
# في مجلد GUI
dotnet test

# أو استخدام Visual Studio Test Explorer
```

### 4. تشغيل جميع الاختبارات
```bash
# في المجلد الرئيسي
make test-all
```

## أنواع الاختبارات

### 1. اختبارات الوحدة
```c
// مثال على اختبار المحلل المعجمي
void testLexer() {
    Lexer* lexer = initLexer("test.arabic");
    
    // اختبار قراءة الكلمات المحجوزة
    Token* token = getNextToken(lexer);
    assert(token->type == TOKEN_PROGRAM);
    
    // اختبار قراءة المعرفات
    token = getNextToken(lexer);
    assert(token->type == TOKEN_IDENTIFIER);
    
    freeLexer(lexer);
}
```

### 2. اختبارات التكامل
```c
// مثال على اختبار التكامل بين المحللين
void testParserIntegration() {
    // اختبار التكامل بين المحلل المعجمي والنحوي
    Lexer* lexer = initLexer("test.arabic");
    Parser* parser = initParser(lexer);
    
    ParseTree* tree = parse(parser);
    assert(tree != NULL);
    
    freeParser(parser);
    freeLexer(lexer);
}
```

### 3. اختبارات النظام
```csharp
[Test]
public void TestCompleteCompilation() {
    // اختبار الترجمة الكاملة
    string sourceCode = "برنامج اختبار;";
    CompilerInterface compiler = new CompilerInterface();
    
    bool result = compiler.CompileFile(sourceCode);
    Assert.IsTrue(result);
    
    string mipsCode = compiler.GetMIPSCode();
    Assert.IsNotEmpty(mipsCode);
}
```

## تغطية الاختبارات

### 1. تغطية الكود
- **المحلل المعجمي**: 95%+
- **المحلل النحوي**: 90%+
- **المحلل الدلالي**: 85%+
- **مولد الكود**: 80%+
- **واجهة المستخدم**: 75%+

### 2. أنواع الأخطاء المختبرة
- **أخطاء معجمية**: أحرف غير صحيحة، كلمات غير معروفة
- **أخطاء نحوية**: بناء جملة غير صحيح، قواعد نحوية خاطئة
- **أخطاء دلالية**: أنواع غير متوافقة، متغيرات غير معرفة
- **أخطاء توليد الكود**: أخطاء في MIPS، مشاكل في التسجيلات

### 3. سيناريوهات الاختبار
- **البرامج البسيطة**: متغيرات، تعيينات، عمليات حسابية
- **البرامج المتوسطة**: شروط، حلقات، إجراءات
- **البرامج المعقدة**: هياكل بيانات مركبة، إجراءات متداخلة
- **البرامج الكبيرة**: ملفات متعددة، مكتبات

## استكشاف الأخطاء

### 1. مشاكل شائعة
- **اختبارات C لا تعمل**: تأكد من تجميع المكتبات
- **اختبارات C# لا تعمل**: تأكد من تثبيت Unity Test Framework
- **اختبارات الأداء بطيئة**: تحقق من إعدادات النظام

### 2. حلول للمشاكل
```bash
# إعادة بناء المشروع
make clean
make

# إعادة تشغيل الاختبارات
make test
```

### 3. الحصول على المساعدة
- راجع ملفات السجل في مجلد `logs/`
- تحقق من إعدادات Visual Studio
- راجع التوثيق في مجلد `Documentation/`

## الخلاصة

نظام الاختبارات هذا يضمن:
1. **جودة عالية**: تغطية شاملة لجميع المكونات
2. **استقرار**: اختبار جميع الحالات الممكنة
3. **أداء**: قياس وتحسين سرعة المترجم
4. **صيانة**: سهولة إضافة اختبارات جديدة
